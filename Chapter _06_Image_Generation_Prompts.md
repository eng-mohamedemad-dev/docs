# Chapter 06 Image Generation Prompts

Use these prompts to generate the Chapter Six images. The first prompt is for one combined Flutter mobile testing inventory image. Authentication, Assignments, Maps, Panic / SOS, Chat, and Media are not separate images; they are modules inside the same Flutter testing image.

## Global Style

```text
Create a clean, modern editorial illustration for a technical graduation book about Flutter mobile app testing.
Style: premium technical book illustration, semi-flat 3D, soft lighting, crisp details, professional and minimal.
Color palette: teal, navy, white, graphite, with small orange accents.
The result should look like a polished book figure, not a marketing poster.
No readable text, no logos, no brand names, no distorted UI labels.
Landscape 16:9, minimum 3120 px width, high resolution.
```

## CH06_FIG_09_Flutter_Mobile_Testing_Inventory_Map.png

```text
Create one combined chapter figure for:
Flutter Mobile Testing Inventory Map.

The image must represent a single Flutter mobile testing inventory for an officer-side mobile application.
Do not create separate images for each feature. Show all tested mobile areas together as connected modules inside one organized testing map.

Main composition:
Place one large central smartphone or tablet-like mobile testing dashboard in the middle.
Around it, arrange six connected feature modules as clean visual panels or nodes:
1. Authentication
2. Assignments
3. Maps
4. Panic / SOS
5. Chat
6. Media

Each module should visually show its expected validation:
- Authentication: secure login, token storage, login state, logout state, profile access.
- Assignments: new assignment, assignment details screen, accept action, arrive action, resolve action, status transitions.
- Maps: location permission, route display, navigation support, map screen, current location.
- Panic / SOS: emergency request, panic confirmation, cancellation state, alert signal.
- Chat: officer chat screen, text messages, quick replies, voice evidence.
- Media: body-cam upload, evidence upload screen, upload feedback.

Add mobile testing evidence details across the whole image:
automated test paths, 38 test files concept, many Dart test definitions concept, pass and fail indicators, bug markers, screenshot thumbnails, log cards, validation checkmarks, and connected evidence nodes.

Make the image feel like a structured inventory map that documents mobile test coverage from the officer app.
Use abstract UI shapes only; no readable UI text.
Use a clean modern technical-book style, semi-flat 3D, teal and navy palette with orange alert accents.
No readable text, no logos, no brand names.
Landscape 16:9, minimum 3120 px width, high resolution.
```
